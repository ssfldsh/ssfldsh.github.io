Go并发：控制多个线程中部分功能的执行顺序

题目一：顺序打印

要求1：现在有三个线程A、B、C，它们的功能分别为打印A、B、C各五次，

要求修改代码：使用同步语句控制线程执行顺序，使得A的打印发生在B之前，B的打印发生在C之前

package main

import (
    "fmt"
    "sync"
    "time"
)

func main() {
    var wg sync.WaitGroup
    wg.Add(3)
    nowBThreadCanRun := make(chan int)
    nowCThreadCanRun := make(chan int)
    go func() {
        // ３、打印A不需要依赖于谁，直接打印即可
        for i := 0; i < 5; i++ {
            fmt.Println("A")
        }
        nowBThreadCanRun <- 1
        // A线程阻塞时，可以执行B线程，体现了多线程的意义
        time.Sleep(1000 * time.Millisecond)
        wg.Done()
    }()
    go func() {
        //　２、在A线程和B线程中使用使用nowBThreadCanRun，使得A线程中的打印操作发生在B线程之前
        <-nowBThreadCanRun
        for i := 0; i < 5; i++ {
            fmt.Println("B")
        }
        nowCThreadCanRun <- 2
        time.Sleep(1000 * time.Millisecond)
        wg.Done()
    }()
    go func() {
        //　２、在B线程和C线程中使用使用nowCThreadCanRun，使得B线程中的打印操作发生在C线程之前
        <-nowCThreadCanRun
        for i := 0; i < 5; i++ {
            fmt.Println("C")
        }
        wg.Done()
    }()
    time.Sleep(100 * time.Millisecond)
    fmt.Println("main thread end")
    // 使用waitGroup阻塞main线程，防止main线程过早执行结束
    wg.Wait()
}

// 不使用多线程也能实现顺序打印ABC的效果，但是因为在print过程中，加入了很多阻塞cpu的操作（如sleep）
// 使得以非并发方式实现的代码运行时间很长：至少大于代码中所有sleep的时间的总和：
// 以多线程方式实现的顺序打印ABC的代码运行时间为1.115s左右，比顺序执行要快，顺序执行的话，3个sleep操作就至少需要执行2.1s

题目二：现在有A、B两个线程，它们的功能分别为打印A、B各5次

现在要求修改代码，使得：

A和B交替打印：

具体规则：打印一个A后才能打印一个B；打印一个B后才能打印一个A;A先被打印。

package main

import (
    "fmt"
    "sync"
    "time"
)

// const声明以及赋值:
const looptime int = 100

func main() {
    var wg sync.WaitGroup
    wg.Add(2)
    start := time.Now()
    // 交替打印A、B，使用1个或者2个cahnnel来实现:
    canPrintFoo := make(chan int, 1)
    canPrintBar := make(chan int, 1)
    // 在A、B两个线程中引入：canPrintA channel使得对A的打印发生在B之前：
    // 在A、B两个线程中引入：canPrintB channel使得对B的打印发生在A之前：
    // 在创建A、B两个线程前先 canPrintA channel里注入数据。
    canPrintFoo <- 0
    go func() {
        for i := 0; i < looptime; i++ {
            <-canPrintFoo
            fmt.Print("foo")
            canPrintBar <- 0
        }
        defer wg.Done()
    }()
    go func() {
        for i := 0; i < looptime; i++ {
            <-canPrintBar
            fmt.Print("bar")
            canPrintFoo <- 0
        }
        defer wg.Done()
    }()
    wg.Wait()
    fmt.Println("the main function has been run:", time.Since(start))
}

多线程程序适合在I/O密集型应用中使用，当一个线程处于I/O状态时，CPU是空闲的，可以被用来执行其他线程。

大部分网络应用都是I/O密集型的，适合使用并发优化。

常见的I/O优化方式还有

同步非阻塞IO： 在一个线程里使用多路复用，同时监听多个连接。多个连接共享一个线程，避免大量线程创建带来的时间和空间开销。

异步非阻塞IO：不需要主动轮询，访问每个连接状态。当被调用方执行完毕式，主动通知调用方。

对于计算密集型任务，适合使用并行优化：

如游戏还有科学计算领域使用GPU并行处理矩阵计算、处理图像数据。

合适的数据结构和算法也能降低程序的复杂度。某些算法的优化空间特别大，如使用动态规划代替暴力搜索，能将某些O(2^n)的算法优化到O(n)。
